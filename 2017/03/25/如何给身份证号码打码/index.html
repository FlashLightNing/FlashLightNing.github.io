如何给身份证号码打码

发表于 2017-03-25   |     |   阅读次数

背景及要求

     身份证照片中包含的敏感信息较多，尤其是身份证号码，能否找到一种技术手段给定较规范的身份证照片后，能输出一张身份证号码打码了的图片。

技术选型

Java+Javacv
     Javacv就是对opencv的封装，底层调用还是用的opencv的库，在服务器使用成本低，不需要安装各种库和配置，只需要几个jar包即可。

方案

     图片灰度化+二值化+腐蚀/膨胀+连通图检测+连通图筛选。
     原理是将身份证号码区域的数字通过腐蚀/膨胀的手段，使得该号码的区域连通在一起，然后根据该区域的特征筛选得到该区域的坐标位置，然后完成打码。

效果图

（只有下方的身份证号码为程序打码，其余区域是在效果图上手动打码避免信息泄漏）

步骤详解

预处理完后的处理

如图：

     前面3个步骤只是对图像的预处理，最重要的2步是腐蚀/膨胀 完得到的图片进行连通区域检测及筛选。连通区域检测的意思就是检测图中哪几块区域是连在一起的。而筛选则是从得到的这么多连通区域中正确的筛选出身份证号码的区域，比如根据长宽比。但是根据长宽比来判断的话，在身份证号下面有一个不知名的更大的区域。所以光靠长宽比是不够的。

4.获取连通图

     EachCount类主要用于存储每个连通区域的起始坐标(x,y)以及该区域的长和宽，用于根据这些信息对每个区域进行筛选

5.对每个区域进行筛选

     先对list进行排序，根据每个区域的长宽比，长宽比或者宽长比大的在前面，小的在后面。考虑宽长比是因为身份证照片可能会是竖着的情况下。

     在这里需要先说明一下：根据上面的图片，我们可以发现，我们是可以获取到整张图片的连通区域的数量的。那其实我们可以依次对前面排序完的list中的每一个区域做以下处理：获取原图中对应连通图的区域，然后对该区域再做灰度后-二值化处理，然后得到该区域里面的连通图的数量，如果该数量等于18（身份证号码的数目），我们就可以认为该区域是身份证号码的区域

     在实际处理中，我遍历list的前5个长宽比最大的区域，超过5个则认为此次处理失败，避免list元素较多时全部遍历降低处理效率。
     对区域进行打码主要用的是高斯模糊进行处理。
